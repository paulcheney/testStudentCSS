Individual Project Site Plan

Blueprint for the Montevideo Chamber website — outlines purpose, audience scenarios, colors, typography, wireframes and testing plan. This document will evolve as development continues.

Site Name

Name
Montevideo Chamber
Reason
This name clearly identifies the local business chamber for Montevideo and is suitable for branding, event promotion and member directory services.
Suggested domain
montevideochamber.uy (preferred local domain)
montevideochamber.org (international fallback)

Site Purpose

The Montevideo Chamber website will provide a community hub for local businesses, event listings, member spotlights, and resources. Key features include:

  • Business directory searchable by category and membership level.
  • Upcoming events feed and event detail pages.
  • Membership application (Join) with multi-level options and modals describing benefits.
  • Member spotlights and featured advertising positions for sponsors.
  • Local weather snippet for Montevideo and contact information for the chamber.

Scenarios (target user questions)

  • Local business owner: "How do I apply to become a member and what are the membership benefits?"
  • Visitor planning an event: "What chamber-sponsored events are upcoming and how do I register?"
  • Resident looking for services: "Which local businesses offer the service I need (location, hours, contact)?"

Color Schema

Primary colors chosen for accessibility, contrast and professional visual identity. These are used throughout the site and styling in this document reflects the palette.

--primary — #0a84ff (buttons, links, accents)
--secondary — #111111 (headings, strong text)
--background — #f5f5f7 (page background)
--success — #34c759 (status, success badges)
Usage
  • --primary: primary CTAs, links, active nav, focus outlines.
  • --secondary: headings, emphasis text, high-contrast body copy.
  • --background: page background, subtle section backgrounds.
  • --card: cards and panels background (white)

Typography

Selected web fonts (kept simple for readability and performance):

Primary font
Inter — used for body text, navigation, form inputs (weights: 300,400,600,700).
Heading font
Merriweather (serif) — used for H1/H2/H3 to add contrast and a formal tone for headings.
Special
System UI / monospace only for small code snippets or data displays if necessary.

CSS / Assets

Planned files and structure (mobile-first):

  • styles/normalize.css — reset/normalization
  • styles/small.css — mobile baseline styles (typography, layout, events card rules)
  • styles/events.css — optional: event-specific styles (if separate)
  • styles/large.css — responsive rules for ≥768px
  • scripts/events.js, scripts/join.js and scripts/weather.js
  • Image assets inside images/ and data inside data/events.json

Testing & Validation

Planned self-audit and tests to run before submission:

  • W3C HTML Validator — valid HTML5 (doctype, lang, meta charset first)
  • CSS Validator — no major errors
  • Accessibility audits (Lighthouse / axe): aim for 90+ accessibility score
  • Lighthouse: Performance, Best practices, SEO (run for mobile + desktop)
  • Color contrast: Ensure WCAG AA for body text and UI controls (contrast checker)
  • Keyboard navigation: Tab order, visible focus states for links, forms and modals
  • Form validation: client-side required attributes + server-side/thankyou page displays submitted required fields
  • DevTools console: fix JavaScript runtime errors

If the automated course audit tool fails, provide a short note in Canvas and reply to the grader with a link and description of the steps you took — include screenshots of Lighthouse results.
